De vragen van de GIP-jury

Wat is voor dokters belangrijk aan IT / Wat verwachten dokters van IT / het netwerk:

  • Welke info moeten dokters ten allen tijde binnen handbereik hebben van een patiënt?
    Puur theoretisch ‘moeten’ ze niks in handbereik hebben, nochtans kunnen ze het beste het medische dossier (allergieën, vorige ziektes) binnen handbereik hebben.

  • Wat moet de informatica / computers / de servers bijhouden en faciliteren als een kind van 10 met een gebroken arm op de spoedafdeling komt?
    ° Foto’s RX in de beeldbank (PACS)
    ° Verslag van onderzoek (Medisch Dossier)
    ° ID-gegevens, Ziekenkas, … (administratieve zaken)

  • Hoe moet die info ingegeven worden, bewaard, bevraagd en hoe wordt deze ter beschikking van behandelende artsen gesteld?
    ° KWS / OAZIS / Medisch Dossier -> Ingave door administratieve verpleging
    ° Bewaard -> Netwerkopslag
    ° Bevraagd -> via pc’s -> toepassingen

Bespreek de back-up- en recoverystrategie.

  • Hoe vaak en hoe worden er back-ups genomen? Van welke data?
    -> Elke dag incremental filesystemen servers oracle cluster hosts en VMWare rdm-filesystemen.
    -> Elke dag transaction log’s en data-bestanden nvan databases
    -> Elke dag backup van snapshots vmdk-files VMWare
    -> Wekelijkse full backup van oracle cluster hosts

  • Recovery: hoe lang “mogen” bepaalde systemen (servers), databases, … “down gaan”?
    -> Onderverdeling in critical / non-critical
    -> Critical mag eigenlijk niet down tenzij gepland en in overleg met end-users
    -> Non-critical kan iets makkelijker onderhoud op gebeuren
    -> Voor alle data geldt dat die op high-availability systemen actief zijn.

  • Welke data, services, … mogen nooit uitvallen (en waarom)?
    -> Het OAZIS patiëntadministratie systeem moet altijd blijven werken omdat hier de bron van de
    administratie en tarificatie gebeurt, zonder dit kan de rest zowiezo niet werken.
    -> Het KWS klinisch werk systeem moet zoveel mogelijk blijven werken omdat hier alle
    patiëntgegevens toegankelijk gemaakt worden.

  • Hoe zorgt de IT er dan voor dat dit niet kan?
    -> IT zorgt voor een infrastructuur die high availability ondersteunt:
    -> Replicatie en mirroring van gegevens binnen SAN storage units.
    -> Een farm van fysieke vsphere-vmware servers die synchroon load balanced werken.

  • Mogelijk bij-onderwerp: LSA (level service agreement)
    SLA: totaal niet van toepassing
